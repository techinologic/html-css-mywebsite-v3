My HTML Cheat Sheet


1. <!DOCTYPE>

Although not a tag, but rather an instruction, this should always be the first line of an HTML code. This tells the browser about what version of HTML is made with so it knows what type of document to expect.


2. <html></html>

This tag is the top-level element of an HTML document, also known as the root element. This defines the HTML document and all elements should be inside this. I think of this as the fence in my backyard.


3. <head></head>

This tag gives information, or to be more specific, metadata (a data that describes data), about the document. Inside the head tag are the title tags, and link tags that link to the CSS files and JavaScript. This is also we I linked my external fonts from google using script.


4. <title> </title>

Inside this tag will be the title of the webpage which can be seen on the browser tab. It can only take in text.


5. <li></li>

This tag is used when making a list, but must be used inside ordered list <ol> or unordered list <ul>. I used this in listing my navigation links (my first version), my future to do lists, and change logs.


6. <a> </a>

We use the anchor tag to define hyperlinks, which brings you to another page, different page locations and send email when clicked. It uses the attribute href which contains the URL where the hyperlink points to. My navigation links are all in anchor tags, that when clicked brings you to the page stated in the link.


7. <body> </body>

The body tag defines the body of a document. This is where every page information will reside and there can only be one body tag for each page.


8. <ul> </ul>

The unordered list is used we want to show an unordered bulleted list. As I stated above, this is used with <li> tag inside.


9. <h1> to <h3>

These are section headings, the lower number means bigger text or a higher level. <h1></h1> are usually the title of the page like “Welcome to my page!” and <h2></h2> are usually the sub-titles and sub-headings.


10. <br>

This is the line break tag which inserts a line break, which moves the flow of content 1 line below. I may have abused this since I use it to make spaces between elements.


11. <em></em>

What is inside the emphasize tag will show italicize, showing emphasis. I prefer the <strong> tag than this one in emphasizing text.


12. <strong></strong>

The strong tag is also a way of emphasizing text. Any text inside the strong tag will show in boldface which I think is a better for emphasizing.


13. <img>

The image tag is used to show an image into the document. Some of the attributes of the image tags are src, which is the location of the image, alt which is a description on the image, for people with disabilities, and the images dimensions width and height which defines how wide and tall the image should appear.


14. <iframe> </iframe>

The inline frame is used for embedding another HTML page into the current page. This is what I used when I embedded my essay files on my project page.


15. <script></script>

This is a dedicated tag for JavaScript which is used to embed or source executable JavaScript code. I used this to reference an external font from google, and in embedding the Google Maps API in my contacts page.


16. <div></div>

The division tag is the generic container for content. By using div tags, we can group contents so we can easily style them in the stylesheets by using the class attribute. We can style each div tags if given separate classes by stating which class.


17. <nav> </nav>

We use nav tags to represent part of the page used primarily in navigation links. Screen readers for disabled users will ignore the texts inside the nav tags.


18. <footer></footer>

The footer tag defines the section at the bottom of the page where information about the author, copyright or other information are contained.


19. <!—comments -->

This element is used to add a comment to an HTML document. Any text inside the tag are not rendered when the HTML document is rendered by a browser.


20. <hr>

The hr element is used as a horizontal separator of content when there is a change of topic in a section. It renders as a horizontal line and because of that I used it for presentation purposes but I know now that I shouldn’t.


21. <p></p>

The paragraph tag represents paragraphs of text, like in the books, where we see paragraphs as blocks of text separated from other paragraphs. Text enclosed in this tag will be formatted and will look like a paragraph.

Source: Descriptions taken from W3Schools's HTML element reference and Mozilla MDN Web Docs.
